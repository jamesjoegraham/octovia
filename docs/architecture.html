🦓 Octovia — Render Pipeline Architecture

A bespoke, DOM-free state diagram rendering engine in pure Rust • 6 phases • 8 ports • 80 tests

Core Principle

Easy to read, easy to write — every graph is beautiful because every node connects through exactly 8 ports. No arbitrary angles, no overlapping chaos, no bezier spaghetti. Output is pure SVG in the London Underground transit-map style.

Pipeline Overview

The engine processes a DSL description through 6 phases, each transforming the AST toward the final SVG. The pipeline is linear — each phase depends only on the output of the previous one.

Phase 1 — Parse

Two input formats: a text DSL (parsed line-by-line with string splitting, not nom combinators) and JSON (deserialised with serde). Both produce the same AST types.

# Comment line title: My State Machine theme: ember Idle -> Active : recheck Active -> Processing : submit Processing -> Done : complete Done -> Idle : reset

Phase 2 — Measure

A global FontSystem (initialised once via OnceLock<Mutex<>>) loads JetBrains Mono at 14px. Each label is measured and text wraps at MAX_NODE_WIDTH = 180px.

// Constants MAX_NODE_WIDTH: 180.0 // wrap limit FONT_SIZE: 14.0 // px LINE_HEIGHT: 1.35 // line spacing NODE_PADDING: 24 // px inside node MIN_NODE_SIDE: 60 // smallest dimension

Phase 3 — Backbone Layout

3a — Spanning Tree Extraction

BFS from the root node (preferring one with zero incoming edges). Forward edges form the happy path. Edges where spanning_index(to) ≤ spanning_index(from) are tagged is_cyclic = true.

Spanning Tree + Back-Edge

Cyclic A* Routing

3b — Boustrophedon Grid

The spanning tree is laid out in a serpentine (ox-ploughing) pattern. Even rows fill left→right, odd rows right→left, wrapping at a row width computed from the viewport.

per_row = max(1, (viewport_width - 4 * node_half_w) / GRID_SPACING_X) // Boustrophedon rule: if row % 2 == 0: x = margin + col * GRID_SPACING_X // left→right else: x = margin + (per_row - 1 - col) * GRID_SPACING_X // right→left

Phase 4 — Edge Routing (The 8-Port Constraint)

The 8-Port System

TopLeft Top TopRight ●──────●──────● │ │ Left ●──┼── [ NODE ] ──┼──● Right │ │ ●──────●──────● BottomLeft Bottom BottomRight Forward edges: East → West (straight horizontal) Cyclic edges: North → South (A*-routed octilinear) Diagonal ports: defined in AST, ready for future assignment

Two-Pass Routing

Pass 1 — Forward edges (deterministic, O(n)):

Every forward edge exits the source's East port and enters the target's West port along a straight horizontal line at the source's Y-coordinate. Zero turns. Zero crossings.

Pass 2 — Cyclic edges (A* with transit-map cost):

// A* cost function f(n) = g(n) + h(n) + T(n) + C(n) g(n) = movement cost from start h(n) = octile distance to goal T(n) = turn_penalty * direction_changes (~200 each) C(n) = cross_penalty * occupied_cells (~500 each) // Movement costs axis-aligned (E/W/N/S): 10 diagonal (NE/NW/SE/SW): 14 // Octile heuristic fn octile_distance(a, b): d = min(|dx|, |dy|) return (|dx| + |dy| - d) * 10 + d * 14

Grid Occupancy

A 10px-resolution grid tracks which cells are occupied by nodes (3-cell-radius square) and previously-routed edges. A* only traverses is_free cells — no overlapping elements.

Phase 5 — Label Placement

Node labels are centred inside the node rectangle. Edge labels sit at the route midpoint, offset 12px above, with opacity="0.7" for visual hierarchy.

The AnchorSlot enum defines all 8 positions around each node: TopBottomLeftRight TopLeftTopRightBottomLeftBottomRight A future collision-resolution pass could score each slot by edge-path intersection count and pick the best one.

Phase 6 — SVG Output

Z-Order (bottom to top)

  1. Background fill (viewport rectangle)
  2. Title text (if any)
  3. Edge paths — stroke-only, dashed for cyclic
  4. Edge labels — midpoint text, 0.7 opacity
  5. Node rectangles — filled + stroked, rounded corners
  6. Node labels — centred inside each node

Auto-Bounding Box

compute_bounds() scans every node position and edge route point, adds 40px padding, and sets viewBox dynamically. The SVG fits its content exactly — no wasted space, no clipping.

Why 8 Ports = Beautiful Graphs

Diamond + Feedback

✅ Planarity — bounded edge density
✅ Ordering — forward uses E/W, cyclic uses N/S
✅ Octilinearity — only 0°/45°/90°/135° angles
✅ Separability — A* always finds clean paths for cycles
✅ Readability — forward edges never turn
